# Classroom support workflow

When something goes wrong in a course setting, the fastest sequence is usually:

1. Confirm everyone is using the same version of the app.
2. Retry the task with the same browser and lesson steps.
3. Check the [Troubleshooting] ideas below guide.
4. For reporting Problems with the app: send email to Avida-ed-development@googlegroups.com

# Troubleshooting

Use this checklist when Avida-ED does not launch or when assignment workflow
steps fail.

## If the app does not launch

- confirm you started from the current [Launch Avida-ED](/app) page
- reload the page once, this sometimes helps with cache proplems
- Try a Hard Refresh
- - Most Browsers: Hold down the Shift or Ctrl key (depending on the browser/OS) and click the Reload button in the address bar.
- - Windows / Linux: Ctrl + F5 or Ctrl + Shift + R
- - macOS: 
- - - Chrome/Firefox: Cmd + Shift + R
- - - Safari: Cmd + Shift + R
- try a current desktop browser if you were using an older or mobile setup
- check whether privacy tools or pop-up blocking interfere with the app

## If a classroom activity behaves unexpectedly

- confirm all students are using the same version
- make sure students are not changing more than one variable at a time
- rerun the same setup once to see whether the pattern is reproducible
- compare the assignment against the current
  [Curriculum](/curriculum), [Lab Manual](/curriculum/lab-manual), or
  [Model Lessons](/curriculum/model-lessons) page if students are using older
  handouts

## If students cannot save or export files

- confirm the device permits downloads
- check whether the browser blocks downloads by default
- verify that the file appears in the expected download location

## When to report an issue

Report the problem when:

- the issue persists after a reload
- it affects multiple users on the same setup
- it prevents completion of a lesson or assignment

Include the details listed on the [Support Docs](/docs/support) page.
For a clean report format, use [Reporting Problems](/docs/support/reporting-problems).

# Reporting Problems 

Every useful report should include:

- which Avida-ED version you were using (such as 4.0.33)
- the browser and operating system
- whether the problem happened on one device or many
- the exact step where the problem appeared
- what you expected to happen
- what actually happened

## Helpful evidence

If your course or institution permits it, include:

- a screenshot of the error or unexpected state
- a short screen recording
- the exact assignment or lesson step that triggered the issue
- a note about whether reloading changed the result

## Fast reporting template

Use this structure:

1. I was using `Avida-ED 4.0.33` or `Avida-ED 3.4.0`.
2. I was working in `browser + version` on `device/OS`.
3. I did these steps: `step 1`, `step 2`, `step 3`.
4. I expected: `expected behavior`.
5. I observed: `actual behavior`.
6. I tried reloading or repeating the steps, and the result `did` or `did not`
   change.

## Classroom triage

Before escalating, check:

- whether everyone in the class is using the same version
- whether the lesson instructions match the version on screen
- whether the issue only appears on one browser or one machine
- whether saving or export permissions are blocked by the local device
