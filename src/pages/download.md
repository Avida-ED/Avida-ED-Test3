title: Download Avida-ED
---

Most users should use the live web version of Avida-ED 4 rather than a local
download.

## Recommended choice

- **Use Avida-ED 4 in the browser** for current teaching and learning work.
- Refer to **Avida-ED 3** only when a course still depends on legacy materials.

## What this page is for

This page is mainly for fallback and archive scenarios. If you do not already
know that you need a download, go back to [Launch Avida-ED](/app) and start
with the current web version.

## When downloads are useful

Download or archive materials are mainly relevant when:

- internet access is limited or unreliable
- a course still uses older instructor handouts or screenshots
- you need historical reference material for earlier Avida-ED versions

## Current links

- [Launch Avida-ED 4](https://avida-ed.github.io/Avida-ED4/)
- [Launch Avida-ED 3](https://avida-ed.github.io/Avida-ED3/AvidaED.html)

## Decision guide

- If you are a student in a current course: use the browser launch unless your
  instructor says otherwise.
- If you are an instructor preparing a class: test the browser workflow first,
  then keep legacy or offline paths only as contingency options.
- If you are matching an older handout or screenshot set: check
  [Version History](/about/version-history) before assuming you need an older
  application version.

## Notes for instructors

If you are preparing a class, test the browser-based workflow before relying on
older downloadable materials. As the new support site is completed, historical
files and legacy instructions should move into clearly labeled archive sections
instead of being mixed with current teaching guidance.

## Legacy offline guidance

The legacy download page included:

- a long-term-use Avida-ED 4 "Anti-BitRot" edition hosted through a separate
  repository
- standalone Avida-ED 3.3.5 packages for Mac and Windows
- Windows localhost instructions for self-hosted classroom use

Those paths are still relevant for fallback scenarios, but they should now be
presented as secondary to the browser-based launch workflow.
