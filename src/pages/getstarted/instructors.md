This page is the instructor-facing starting point for using Avida-ED in class.

Use it when you need to choose a launch path, decide how much scaffolding your
students need, and move quickly into the most relevant curriculum materials.

## Start here

- Confirm that the current app opens from the [Launch Avida-ED](/app) page.
- Review the main [Curriculum](/curriculum) hub for the teaching pathways now
  published in the page hierarchy.
- Use [Quick Start](/curriculum/quick-start) if you want a short first use or
  a low-prep class session.
- Use the [Lab Manual](/curriculum/lab-manual) if students need a more guided,
  sequential introduction.
- Use [Model Lessons](/curriculum/model-lessons) when you want a
  concept-centered activity inside an existing unit.

## Choose the right path

### Quick classroom trial

Best when you want to test Avida-ED in one class period with minimal setup.

- launch the app in advance
- define a simple deliverable
- choose one comparison question
- send students to [For Students](/docs/students) for orientation and workflow

### Guided multi-step sequence

Best when students are new to the environment and need structure.

- begin with [Lab Manual](/curriculum/lab-manual)
- assign Exercise 1 as the common starting point
- add later exercises only if the class needs more practice with evidence and
  comparison

### Concept-focused lesson

Best when your course already has its own structure and you need one targeted
activity.

- use [Model Lessons](/curriculum/model-lessons)
- choose the lesson that matches the concept you are teaching
- review any legacy notes on screenshots or interface wording before class

## Before class

1. Test [Avida-ED 4](/app) on the same device type your students will use.
2. Decide what students will save, export, or submit.
3. Identify whether students need the student workflow pages, a transcript
   page, or troubleshooting guidance in advance.
4. Keep the [Support](/support) page nearby for classroom issues.
